# 🎬 v1.5.0 - Yippie-Ki-Yay

Hello everyone!

Yippie-Ki-Yay! This has been one of the biggest releases to date — over 150 Pull Requests have been merged since v1.4.0, bringing entirely new attack capabilities, new modules across every protocol, and a range of quality-of-life improvements. Whether you are doing internal assessments, CTF challenges, or red team operations, this release has something new for you.

A huge thank you to all contributors who submitted Pull Requests, reported issues, and helped with discussions on GitHub and our [Discord Server](https://discord.gg/pjwUTQzg8R). This community never stops delivering.

If you'd like to contribute to this wiki, the source code is available on GitHub [here](https://github.com/Pennyw0rth/NetExec-Wiki).

## RDP Command Execution

The RDP protocol can now be used to execute commands on a target! Thanks to [@Adamkadaban](https://x.com/Adamkadaban), NetExec can leverage RDP for remote command execution, opening up yet another avenue for interacting with hosts when SMB or WinRM are not available or filtered.

## Raw NTDS/SAM/LSA Dump

Is AV blocking access to the SAM, SECURITY, or SYSTEM registry hives? With the new `ntds-dump-raw` module by [@0xb11a1](https://github.com/0xb11a1), NetExec mounts the disk and parses its raw content to extract SAM, LSA, and NTDS secrets — bypassing AV entirely. The module is also available for the WMI and WinRM protocols.

You can also target specific secrets by using the `TARGET` option to dump only LSA or SAM hashes. The documentation for this module can be found [here](../smb-protocol/obtaining-credentials/dump-ntds.dit.md).

(add-screenshot-here)

## NTDS Password History & AES Keys

The `--ntds` flag now supports a `--history` option, added by [@KriyosArcane](https://github.com/KriyosArcane) and implemented with [@azoxlpf](https://github.com/azoxlpf) [@NeffIsBack](https://x.com/al3x_n3ff). In addition to NT hashes, you can now dump the AES Kerberos keys and the full password history of domain users directly from the Domain Controller.

(add-screenshot-here)

## Kerberoasting Without Credentials

You can now perform Kerberoasting **without valid domain credentials**! Thanks to [@azoxlpf](https://github.com/azoxlpf), if you have access to an account that does not require Kerberos pre-authentication, you can use it to request TGS tickets for any kerberoastable service — no password needed.

Additionally, Kerberoasting now supports targeting **specific users** ([@Marshall-Hallenbeck](https://x.com/MJHallenbeck)) and **computer accounts** ([@NeffIsBack](https://x.com/al3x_n3ff)).

(add-screenshot-here)

## ASREPRoast Without Authentication

ASREPRoasting no longer requires a valid user account either. [@NeffIsBack](https://x.com/al3x_n3ff) implemented support for running ASREPRoast with anonymous LDAP logins, so you can enumerate AS-REP roastable accounts and retrieve their hashes without any credentials at all.

(add-screenshot-here)

## Raisechild — Cross-Forest Privilege Escalation

The new `raisechild` module by [@azoxlpf](https://github.com/azoxlpf) automates cross-forest privilege escalation. If you have compromised a child domain, this module handles the full escalation to the parent domain automatically. The wiki page for this module is available [here](../ldap-protocol/raisechild.md).

(add-screenshot-here)

## Certipy Module

ADCS remains one of the most impactful attack surfaces in Active Directory environments. The new `certipy-find` module by [@NeffIsBack](https://x.com/al3x_n3ff) integrates the `certipy find` command directly into NetExec, giving you an instant overview of all certificate templates and their (mis)configurations without leaving the tool.

(add-screenshot-here)

## Entra ID Sync Server & Credentials

If you want to pivot into Entra ID (formerly Azure AD), NetExec now has two new capabilities from [@NeffIsBack](https://x.com/al3x_n3ff):

- The `entra-id` module discovers the Entra ID sync server running in the environment.
- The `msol` module then dumps the sync credentials from that server, giving you a direct path into the cloud tenant.

The documentation can be found [here](../ldap-protocol/enumerate-entra-id.md).

(add-screenshot-here)

## BadSuccessor Module

The `badsuccessor` module by [@mpgn](https://x.com/mpgn_x64) checks for the BadSuccessor vulnerability in the environment. Checking for this newly disclosed attack path is now as simple as running a single NetExec command.

(add-screenshot-here)

## CVE-2025-33073 — NTLM Reflection Check

A new SMB module `ntlm_reflection` by [@Mauriceter](https://github.com/Mauriceter) checks whether the NTLM reflection attack (CVE-2025-33073) has been patched on the target. Relaying SMB traffic to LDAP is still a powerful technique — now you can quickly verify which hosts remain vulnerable.

(add-screenshot-here)

## dns-nonsecure Module

The new `dns-nonsecure` module by [@MaxToffy](https://github.com/MaxToffy) automatically discovers DNS zones where records can be added **without any authentication**. If such zones exist in the environment, an attacker can add arbitrary DNS records to redirect traffic or facilitate further attacks.

(add-screenshot-here)

## MSSQL SAM & LSA Dump

Do you have admin privileges on an MSSQL server? NetExec can now dump SAM and LSA secrets directly via the MSSQL protocol with the new `--sam` and `--lsa` flags, implemented by [@NeffIsBack](https://x.com/al3x_n3ff). No need to switch protocols.

(add-screenshot-here)

## Session Enumeration Without Admin Privileges

Until now, enumerating active sessions with NetExec required administrative access to the target. With the new `--reg-sessions` flag by [@MaxToffy](https://github.com/MaxToffy), sessions can be enumerated using normal user privileges via the remote registry service. Documentation is available [here](../smb-protocol/enumeration/enumerate-logged-on-users-winreg.md).

(add-screenshot-here)

## change-password Module

`STATUS_PASSWORD_MUST_CHANGE` and `STATUS_PASSWORD_EXPIRED` will no longer block your access. The new `change-password` module by [@KriyosArcane](https://github.com/KriyosArcane) and implemented with [@termanix](https://github.com/termanix), [@NeffIsBack](https://x.com/al3x_n3ff). Lets you reset the password of an account in this state so you can continue working with it immediately. Documentation is available [here](../smb-protocol/change-user-password.md).

(add-screenshot-here)

## Recyclebin Module

The admin finally deleted `password.txt` from the desktop? If the Recycle Bin was not emptied, you may still be in luck. The new `recyclebin` module by [@Dfte](https://x.com/Defte_) scans the Recycle Bin for files that might still contain sensitive data.

(add-screenshot-here)

## EventLog Credentials

If process creation auditing is enabled on a target, credentials typed as command-line arguments can end up in the Windows Event Log. The `eventlog_creds` module by [@lodos2005](https://github.com/lodos2005) (improved by [@NeffIsBack](https://x.com/al3x_n3ff)) automatically searches through event log entries to recover these credentials. Documentation is available [here](../smb-protocol/obtaining-credentials/eventlog-creds.md).

(add-screenshot-here)

## LDAP get-info-users Module

Passwords can end up anywhere in LDAP — including the `info` attribute of user objects, which is commonly used for notes. The new `get-info-users` module by [@sepauli](https://github.com/sepauli) queries all user objects and surfaces any content in this field that might contain credentials.

(add-screenshot-here)

## GPO Deployed Privilege Assignments

The new SMB module by [@Yeeb1](https://x.com/Yeeb_) extracts privilege assignments that have been deployed via Group Policy Objects. This makes it easy to identify which users or groups have been granted sensitive privileges (such as `SeDebugPrivilege` or `SeTakeOwnershipPrivilege`) across the domain through GPO.

(add-screenshot-here)

## AWS Credentials Finder

The new `aws-credentials` module by [@dev-fortress](https://github.com/dev-fortress) scans the target for AWS credential files and configuration that may have been left on Windows machines by developers or administrators. Finding cloud credentials during an internal assessment has never been easier.

(add-screenshot-here)

## Certificate Request via schtask_as

The `schtask_as` module has received a powerful upgrade from [@Dfte](https://x.com/Defte_). Instead of just executing commands in the context of another user, you can now request a certificate in their context. This allows you to directly impersonate the target user on any machine in the domain, without needing to know their password.

Also in this release: `schtask_as` can now upload and execute custom binaries on the target, with full control over the command and binary used. Documentation is available [here](../smb-protocol/command-execution/execute-remote-command/README.md).

(add-screenshot-here)

## drop-library-ms Module

The new `drop-library-ms` module by [@XedSama](https://github.com/XedSama) implements yet another technique to force Windows into sending credentials to an attacker-controlled server — this time via `.library-ms` files. A useful additional coercion method during internal assessments.

(add-screenshot-here)

## Notepad Module

Unsaved content in the standard **Windows Notepad** can leave traces on disk. The new `notepad` module by [@termanix](https://github.com/termanix) extracts the contents of these unsaved Notepad files from the target, similar to the `notepad++` module from v1.4.0. Documentation is available [here](../smb-protocol/obtaining-credentials/dump-notepad.md).

(add-screenshot-here)

## WinRM Credential Manager Dump

NetExec can now dump credentials stored in the Windows Credential Manager via the WinRM protocol. Thanks to [@tiagomanunes](https://github.com/tiagomanunes), DPAPI-protected credential manager entries can be extracted through an authenticated WinRM session.

(add-screenshot-here)

## Tasklist Filtering & Taskkill

The `--tasklist` flag, introduced in v1.4.0, now supports **filtering** by process name so you can quickly find the PID you need without sifting through hundreds of entries. On top of that, [@Dfte](https://x.com/Defte_) has added a `--taskkill` option — if your payload hangs, you can now kill the process remotely without any additional tooling. `--qwinsta` also supports filtering by a specific username.

(add-screenshot-here)

## Host Banner Improvements

The host banner now surfaces three additional pieces of information automatically:

- **Null authentication**: If the server allows authentication without credentials, it is flagged directly in the banner. Implemented by [@NeffIsBack](https://x.com/al3x_n3ff).
- **LDAP Signing & Channel Binding**: The previously separate LDAP checker has been removed. LDAP signing and channel binding requirements are now displayed natively in the host banner, thanks to [@zblurx](https://x.com/_zblurx).
- **MSSQL Encryption Requirements**: For MSSQL targets, encryption requirements are now shown in the host banner automatically. Implemented by [@Dfte](https://x.com/Defte_).

(add-screenshot-here)

## BloodHound Community Edition Support

BloodHound Community Edition (CE) is taking over as the standard, and NetExec now supports its collector format. If you prefer the legacy collector, you can still switch back via a config setting. Thanks to [@mpgn](https://x.com/mpgn_x64) for the integration.

(add-screenshot-here)

## DC-List with Trusted Domains

The `--dc-list` flag now enumerates Domain Controllers across **trusted domains** as well, not just the current domain. This was implemented by [@termanix](https://github.com/termanix) and makes cross-domain enumeration significantly more streamlined. Documentation is available [here](../ldap-protocol/dc-list.md).

(add-screenshot-here)

## Presence Module

The new `presence` module by [@crosscutsaw](https://github.com/crosscutsaw) and [@NeffIsBack](https://x.com/al3x_n3ff). It checks for artifacts left behind by admins who are not following a tiered infrastructure model — for example, signs of admin accounts being used on workstations where they should not appear. A useful check during security assessments.

(add-screenshot-here)

## MSSQL --database Option

The MSSQL protocol now has a built-in `--database` option by [@A3-N](https://github.com/A3-N), allowing you to specify a target database directly for enumeration commands. This simplifies working with specific databases on a server without additional SQL commands.

(add-screenshot-here)

## PSO as a Core Feature

The Password Settings Objects (PSO) lookup, previously available as a module, has been promoted to a **core LDAP feature** by [@mpgn](https://x.com/mpgn_x64). Fine-grained password policies can now be queried directly without loading a module. Documentation is available [here](../dump-pso.md).

## XDG Base Directory Support

NetExec now respects the [XDG Base Directory Specification](https://specifications.freedesktop.org/basedir-spec/latest/), thanks to [@d4ytox](https://github.com/d4ytox). Configuration files and data are stored in the proper XDG directories on Linux, making NetExec behave like a well-behaved citizen on the system.

# 🩹 v1.5.1 - Patch Release

## Security Fix: spider\_plus Arbitrary File Write

{% hint style="danger" %}
**All users should upgrade to v1.5.1 immediately.** This release patches a security vulnerability in the `spider_plus` module that allowed arbitrary file writes. A huge thank you to [@RaynLight](https://github.com/RaynLight) for reporting this responsibly and professionally.
{% endhint %}

The fix was implemented by [@Marshall-Hallenbeck](https://x.com/MJHallenbeck) in [PR #1121](https://github.com/Pennyw0rth/NetExec/pull/1121). You can upgrade by installing from GitHub via `pipx`:

```bash
pipx install git+https://github.com/Pennyw0rth/NetExec.git
```

## ShadowCopy List Snapshots

The `shadowcopy` module now has a `list-snapshots` action, thanks to [@XiaoliChan](https://x.com/Memory_before). You can now enumerate all existing Volume Shadow Copy snapshots on a target over both the SMB and WMI protocols — useful for identifying snapshot-based credential recovery opportunities.

(add-screenshot-here)

## Notable Fixes

- **Pre2k locked accounts**: Changing the password for locked pre-Windows 2000 computer accounts is now supported ([@NeffIsBack](https://x.com/al3x_n3ff)).
- **FTP hidden files**: `--ls` on FTP now correctly lists hidden files (dotfiles) ([@strikoder](https://github.com/strikoder)).
- **Group user listing**: Fixed an issue where not all users were listed when enumerating groups ([@termanix](https://github.com/termanix)).
